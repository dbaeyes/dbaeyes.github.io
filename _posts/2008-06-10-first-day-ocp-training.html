知识小记：

RMAN
1. RMAN can backup: data files, control files archived redo logs, spfile. NOT the redo log files

2. ORACLE 自身并不支持tape，需要第三方的软件。 oracle自己有SECURE BACKUP软件

3. RMAN中的RETENTION POLICY 设置要小于等于 控制文件的 CONTROL FILE RECORD KEEP TIME

4. BACKUP RECOVER AREA 只能备份到tape上

5. RMAN> BACKUP DATABASE PLUS ARCHIVELOG;
如果备份是是 archived redo log files不能包含其他类型的文件

6. list backup;
list backup of database;
list copy;
list copy of database archivelog from time = 'sysdate -7';

7. report need backup recovery window of 7 days;
report need backup redundancy 2;

report obsolete;
report obsolete redundancy 2;

delete obsolete redundancy 2;
delete backup;

8. V$PROCESS, V$SESSION, V$SESSION_LONGOPS

做过rman备份后，如果丢失了控制文件，数据库只能启动到nomount模式，那么可以使用rman恢复备份的控制文件

RMAN>restore controlfile from autobackup;

SQL> alter database mount;

RMAN> recover database; -- 如果是创建了新的控制文件，需要 USING BACKUP CONTROFILE;

SQL> alter database open resetlogs;
